How It Works

Repo in. Worktrees out. Code done.

Each task lives in its own branch and directory. Nothing touches your working tree.

1

Point it at a repo

Select any local git repo. Branches and remotes are picked up automatically.

2

Branch off a worktree

Hit ⌘N and OpenForge creates a fresh git worktree on its own branch. Describe what you want built.

3

Let Claude work, then merge

Review the diff, browse files, revert if needed. Merge when you're happy.

Features

Git worktrees meet Claude Code.

See every change, in every branch, as it happens.

Real git worktrees

Not simulated sandboxes — actual git worktree directories, each on its own branch. Your main branch stays clean.

Many Claudes, many branches

Each worktree gets its own Claude Code session with full multi-turn history. Five features at once, zero context switching.

Git-backed undo

Before every prompt, OpenForge snapshots the worktree as a git ref. One click rolls back to any previous state.

Diff, files, and terminal

The right panel shows a live diff against the base branch, a full file tree, and an integrated terminal — all per worktree.
